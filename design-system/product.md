# 제품 제약 & 패턴

## B2B SaaS 제약

| 제약 | 규칙 |
|------|------|
| 정보 밀도 | 장식 최소화. 같은 면적에 더 많은 정보 |
| 오류 방지 | 비가역 행동(삭제·전송)은 항상 확인 단계 |
| 반복 효율 | 키보드 내비게이션 필수. 자주 쓰는 행동 = 최소 클릭 |
| 일관성 | 같은 기능 = 같은 위치·모양. 컴포넌트 용도 이탈 금지 |
| 데이터 우선 | Table·Form·Filter 최우선. Empty·Loading·Error state 필수 |

### Density 모드

데이터 밀도가 높은 화면(Table, Form, List)은 density 모드를 선택할 수 있다.

| 모드 | 컴포넌트 height | 사용처 |
|------|-----------------|--------|
| Compact | `--height-32` | 데이터 테이블, 관리자 화면 |
| Comfortable | `--height-36` (기본) | 일반 화면 |
| Spacious | `--height-40` | 폼 중심 화면, 설정 페이지 |

> ⚠️ 한 페이지 내 density 혼용 금지. 페이지 단위로 통일.
> ⚠️ Density 변경은 height 토큰만 바뀐다. 폰트·radius는 그대로 유지.

## 상태 패턴 (Empty · Loading · Error)

데이터를 다루는 모든 컴포넌트는 4가지 상태를 정의한다: **default · empty · loading · error**.

### Empty State

원인별로 메시지와 액션이 다르다. 같은 빈 화면처럼 처리하지 않는다.

| 종류 | 메시지 | 액션 |
|------|--------|------|
| 첫 진입 (데이터 없음) | "아직 [항목]이 없어요" | 생성 CTA |
| 필터 결과 없음 | "조건에 맞는 [항목]이 없어요" | 필터 초기화 |
| 권한 없음 | "이 [항목]에 접근 권한이 없어요" | 관리자 문의 안내 |

### Loading State

| 종류 | 사용처 | 기준 |
|------|--------|------|
| Skeleton | 레이아웃 예측 가능 (Table, Card list, Form) | 1초 이상 예상 |
| Spinner | 예측 불가, 짧은 작업 (버튼 내부, 인라인) | 1–3초 |
| Progress bar | 진행률 표시 가능한 긴 작업 (업로드, 일괄 처리) | 3초 이상 + 진행률 측정 가능 |

### Error State

| 종류 | 사용처 |
|------|--------|
| Inline | 단일 필드 에러 (입력 검증) |
| Banner | 섹션 단위 에러 (저장 실패, 권한 부족) |
| Page | 전체 페이지 로드 실패 (404, 500) |

### 메시지 작성 규칙

> ✅ "네트워크 연결이 끊겼어요. 잠시 후 다시 시도해주세요."

> ❌ "죄송합니다, 알 수 없는 오류가 발생했습니다."

> ⚠️ 모든 에러 메시지는 **원인 + 해결 방법** 구조. 사과·자조 톤 금지.
> ⚠️ Loading 1초 미만은 표시하지 않는다(깜빡임 방지).

## 데이터 포맷팅

표시 형식은 시스템 전체에서 통일한다. 같은 데이터 종류는 한 화면 내에서 형식을 맞춘다.

| 종류 | 형식 | 예 |
|------|------|-----|
| 숫자 | 천단위 콤마, 음수는 마이너스 부호 | `1,234,567` `-128` |
| 큰 숫자 | 한국어 단위 표기 | `1,234만` `12.5억` |
| 소수점 | 항목별 자릿수 통일 | 통화 0자리, 비율 1자리 |
| 날짜 | `YYYY.MM.DD` | `2025.06.01` |
| 날짜+시간 | `YYYY.MM.DD HH:mm` | `2025.06.01 14:30` |
| 상대시간 | 24시간 이내만 사용 | `방금 전`, `3분 전`, `2시간 전` |
| 통화 | 단위 뒤. 천단위 콤마 | `12,000원` |
| 빈값 | 모든 컴포넌트 동일 | `—` (em dash) |
| 진행률 | 정수 % | `85%` |
| ID·코드 | 고정폭 폰트 | `--font-family-mono` |

> ⚠️ 같은 컬럼·같은 데이터 종류는 형식 통일. `1,234`와 `1234` 혼용 금지.
> ⚠️ 24시간 초과 시 상대시간 → 절대 날짜로 전환. (`3일 전` 금지, `2025.05.29` 사용)

## Microcopy & Voice

톤은 **해요체** 기반. 짧고 명확하게.

### 버튼 라벨

| 형식 | 사용처 |
|------|--------|
| 동사 (`저장`, `삭제`, `취소`) | 1차 액션. 기본 |
| 동사+명사 (`항목 추가`, `필터 초기화`) | 대상이 모호할 때만 |

> ⚠️ "저장하기" "확인하기" 같은 ~하기 형태 금지. 명사형으로 통일.

### 다이얼로그·확인

| 상황 | 형식 |
|------|------|
| 비가역 행동 | "[대상]을 삭제할까요?" + 액션 동사("삭제" / "취소") |
| 저장 성공 | 명시적 확인 다이얼로그 없이 toast 피드백 |
| 변경사항 손실 위험 | "변경사항이 저장되지 않아요. 나갈까요?" |

### 에러 메시지

원인 → 해결 방법 순서. 사용자 잘못으로 단정하지 않는다.

> ✅ "이메일 형식이 올바르지 않아요. 예: name@company.com"

> ❌ "잘못된 이메일을 입력하셨습니다."

> ✅ "파일 크기는 10MB 이하만 업로드할 수 있어요."

> ❌ "파일이 너무 큽니다."

### 전문 용어

- B2B 표준 용어는 영어 그대로: 필터, 대시보드, 리포트, API, Webhook
- 일상화된 용어는 한글: 저장, 검색, 설정, 삭제
- 신규 용어 도입 시 한글 우선. 영어 병기는 첫 등장 시 1회만.
